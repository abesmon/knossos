# VRWeb — сигнальный сервер

Минимальный WebSocket-релей для WebRTC-handshake. Группирует подключения по комнатам
(комната = страница, ключ — `PageFetcher.seed_key(url)` на клиенте) и пересылает между
пирами одной комнаты offer / answer / ICE-кандидаты. После установки p2p-соединения
позиции и чат идут напрямую (mesh) — сервер в обмене данными не участвует.

## Запуск локально

```bash
cd signaling
pip install -r requirements.txt
python server.py
```

Сервер слушает `ws://0.0.0.0:8080`. Переменные окружения: `HOST`, `PORT`.

## Запуск в Docker

```bash
cd signaling
docker build -t vrweb-signaling .
docker run -p 8080:8080 vrweb-signaling
```

В клиенте (экран настроек VRWeb) укажите адрес, например `ws://localhost:8080` или
`ws://your-host:8080`.

## Протокол

JSON-сообщения поверх WebSocket.

Клиент → сервер:

| сообщение | смысл |
|---|---|
| `{"type":"join","room":<str>,"nick":<str>}` | войти в комнату |
| `{"type":"offer"\|"answer"\|"candidate","to":<peer_id>,"data":<any>}` | переслать сигнал пиру |

Сервер → клиент:

| сообщение | смысл |
|---|---|
| `{"type":"welcome","id":<peer_id>}` | присвоенный id |
| `{"type":"peers","peers":[{"id","nick"},...]}` | кто уже в комнате (новичку) |
| `{"type":"peer_join","id","nick"}` | вошёл новый пир |
| `{"type":"peer_leave","id"}` | пир вышел/отключился |
| `{"type":"offer"\|"answer"\|"candidate","from":<peer_id>,"data":<any>}` | релей сигнала |

**Антиглар.** Offer создаёт пир с меньшим `id` — так ровно одна сторона инициирует
соединение, без коллизий.

## Федерация (точка расширения, пока не реализована)

`room` — произвольная строка, состояние комнат изолировано. Чтобы пиры с разных
сигнальных серверов попадали в одну комнату (как в Matrix), сервер можно научить
пересылать `join` и сигналы дружественным серверам из конфигурации. peer_id тогда
нужно сделать глобально уникальными (префикс сервера). Это позволит людям хостить
свои сигнальные серверы и обходить блокировки. См. `docs/multiplayer.md`.
