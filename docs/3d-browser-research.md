# 3D Браузер для существующего веба: исследование

> Идея: автоматически превращать произвольные существующие HTML-сайты в навигируемые 3D-пространства — комнаты, острова, мостики — по которым можно перемещаться.

---

## Что уже существует

### 1. "Выдавливание" DOM (не то)

Проекты которые берут DOM и добавляют ось Z — это визуализация структуры страницы, а не навигируемое пространство.

- **Tilt** (Firefox extension, ~2011–2013) — WebGL-расширение, строило 3D-представление DOM: каждый узел как коробка, глубина по Z = глубина в DOM-дереве.
- **dom3d** (bookmarklet, Orion Reed) — рекурсивно добавляет `translateZ(20px)` каждому дочернему элементу, накапливая отступ.
- **Microsoft Edge DevTools — 3D View** — встроен в браузер, три режима: Z-index, DOM-вложенность, Composited Layers.

### 2. "Браузер внутри VR" (не то)

Обычная 2D страница вешается как панель на стену в 3D пространстве. Страница остаётся плоской.

- **Mozilla Hubs** — open source платформа для 3D-пространств в браузере, без VR-шлема.
- **Spatial.io** — коллаборативные 3D-пространства для контента, NFT-галерей и т.д.

### 3. JanusXR — ближайший аналог

**JanusXR / JanusVR** — наиболее близкий к идее проект.

- Переосмысляет веб-страницы как коллаборативные 3D-пространства, соединённые **порталами**.
- Мультиплеер, аватары, работает в браузере без шлема.
- Собственный язык разметки **JML (Janus Markup Language)** — `<FireBoxRoom>`, `<Assets>`, `<Room>`.

**Ключевое ограничение:** чтобы сайт стал 3D-миром, разработчик должен вручную добавить JML-разметку. Произвольный чужой сайт автоматически не трансформируется.

**Site Translators** — частичное решение: JS-скрипты (`example.com.js`), которые говорят Janus как интерпретировать конкретный домен как 3D-пространство. Но это ручная работа под каждый домен.

### 4. JSAR

Open-source spatial browser engine, переосмысляет каждый HTML-элемент как 3D-сущность. Ориентирован на XR-устройства, требует специальной вёрстки.

## Незанятая ниша

| Возможность | JanusXR | Идея |
|---|---|---|
| Произвольные существующие сайты | ❌ нужна JML или ручной транслятор | ✅ |
| Автоматическая трансляция | ❌ | ✅ |
| Навигируемые комнаты / острова | ✅ | ✅ |
| Порталы между страницами | ✅ | ✅ |
| Мультиплеер | ✅ | — |

**Вывод:** никто не реализовал *универсальный автоматический* алгоритм трансляции произвольного HTML в навигируемое 3D-пространство. Site Translators в JanusXR — это ручная реализация того, что нужно сделать эвристически и универсально.

## Алгоритмы трансляции HTML → 3D пространство

### Семантика → топология пространства

HTML5-теги дают готовую структуру для маппинга:

| HTML-элемент | 3D-аналог |
|---|---|
| `<main>`, `<article>`, `<section>` | отдельные комнаты |
| `<nav>` | коридор или хаб с дверьми |
| `<header>` | вход / фасад |
| `<aside>` | боковое ответвление |
| `<footer>` | подвал (буквально) |
| `<a href="...">` внутренняя | дверь в другую комнату |
| `<a href="...">` внешняя | портал в другой мир |

Fallback для сайтов без семантики (div-soup) — визуальные эвристики по размеру, позиции и плотности контента блоков.

### Контент → декорирование

| Контент | 3D-аналог |
|---|---|
| `<img>` | картины / текстуры на стенах |
| Текстовые блоки | таблички, экраны, книги |
| `<h1>`–`<h3>` | масштаб и размещение (h1 над входом) |
| `<video>` | экраны / телевизоры |
| `<form>` | интерактивные панели |

### Граф ссылок → топология между страницами

Внутренние ссылки образуют граф, из которого строится пространство:

- **Слабосвязанные части** → острова
- **Частые переходы** → мостики
- **Страницы с высоким in-degree** → большие центральные залы-хабы

### Визуальный вес → размер пространства

Рендер страницы через Puppeteer/Playwright + анализ скриншота: насыщенность контентом блока определяет размер его "комнаты".

## Технические проблемы

### X-Frame-Options / CSP
Большинство сайтов блокируют встраивание через iframe. Варианты обхода:
- Собственный браузерный движок поверх Chromium (Electron + CDP)
- Расширение для браузера с доступом к живому DOM
- Работа со снапшотами DOM / скриншотами

### CSS не имеет Z-buffer
Браузер использует правила stacking context вместо Z-buffer — возможны визуальные артефакты при пересечении 3D-элементов. Решение: контроль DOM-порядка и стратегический z-index.

### Div-soup
Огромное количество сайтов написано без семантической разметки. Нужны визуальные эвристики как fallback.

### Производительность
Контент сайта внутри 3D-комнат: либо canvas-текстуры (статично), либо CSS3D planes (интерактивно, но с ограничениями).

## Потенциальные строительные блоки

- **Three.js / Babylon.js** — 3D-рендер в браузере
- **JanusXR Site Translators API** — хорошая основа для прототипа транслятора
- **Puppeteer / Playwright** — headless рендер для анализа страниц
- **Electron + CDP** — доступ к DOM без ограничений CSP
