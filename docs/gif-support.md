# Поддержка GIF (анимированных и статичных)

> **Суть:** в Godot нет встроенного декодера GIF, а наши картинки приходят из сети/ФС в
> рантайме (их нельзя предподготовить импортом). Поэтому декодируем GIF сами — на чистом
> GDScript — и многокадровые отдаём как самопроигрывающийся `AnimatedTexture`.

---

## Зачем свой декодер

`Image` в Godot умеет png/jpg/webp/svg, но **не GIF**. Варианты были:

- **GDExtension-аддон** поверх C-библиотеки (giflib/stb) — пришлось бы собирать нативный
  бинарь под каждую платформу экспорта. Лишняя зависимость и боль сборки.
- **Свой декодер на GDScript** — ноль внешних зависимостей, работает везде, где работает
  Godot. GIF из веба маленькие, LZW в GDScript тянет легко. **Выбрали это.**

Ключ: импорт-пайплайн Godot тут не при чём — GIF приходит как байты в рантайме (как и
остальные картинки страницы), поэтому решение живёт целиком в коде загрузки.

## Как это устроено

```
HTTP/FileAccess → байты GIF → ImageLoader._decode (hint=="gif")
    → ImageLoader._decode_gif → GifDecoder.decode(bytes) → Array[{image, delay}]
        1 кадр  → ImageTexture            (как обычная статичная картинка)
        N кадров → AnimatedTexture          (Godot проигрывает сам)
```

`AnimatedTexture` наследует `Texture2D`, поэтому **без единой правки потребителей**
(`ImagePanel._on_texture`, `vrweb_ext_injector`) встаёт в `albedo_texture` и крутится сам.
Кэш `ImageLoader` по URL держит одну текстуру — несколько панелей с одной гифкой играют
синхронно, общей памятью.

### `GifDecoder` (scripts/gif_decoder.gd)

Чистый GDScript, статический `decode(bytes) -> Array`, по кадру на элемент:
`{ "image": Image(RGBA8, размер логического экрана), "delay": float (секунды) }`.

Что разбирает:
- **Логический экран** + глобальная палитра (GCT).
- **Graphic Control Extension**: задержка кадра (центисекунды → секунды), прозрачный индекс,
  метод disposal. Действует на ближайший следующий кадр.
- **Image Descriptor**: позиция/размер кадра, локальная палитра (LCT), interlace-флаг.
- **LZW-распаковку** (`_lzw_decode`) — переменный размер кода, словарь до 12 бит, обработка
  clear/end-кодов и особого случая KwKwK.
- **Чересстрочность** (`_interlaced_row`) — 4 прохода (8/8, 4/8, 2/4, 1/2).

**Composition.** Каждый кадр GIF может быть частичным (дельта поверх предыдущего) с
прозрачностью и методом disposal. Декодер держит общий RGBA-холст и компонует на него:
- disposal 0/1 — оставить как есть;
- disposal 2 — очистить прямоугольник прошлого кадра в прозрачность (restore to background);
- disposal 3 — восстановить холст к снимку до прошлого кадра (restore to previous).

Каждый выходной `Image` — уже самодостаточный полный кадр (снимок холста), его можно отдать
в текстуру как есть. Без этой композиции «оптимизированные» гифки превращаются в кашу.

### `ImageLoader._decode_gif`

- 1 кадр → `ImageTexture` (статичный GIF дёшев, как обычная картинка).
- N кадров → `AnimatedTexture`: `frames`, по кадру `set_frame_texture` + `set_frame_duration`.
- Кадры ужимаются до **512 px** по большей стороне (анимация × N кадров — это память,
  поэтому жёстче обычного потолка 1024 для статичных).
- `AnimatedTexture` держит максимум **256 кадров** — длинные гифки подрезаются.
- Пустой/битый поток → `null`, картинка получает заглушку (как и для других форматов).

## Ограничения

- **Память.** Каждый кадр — полная RGBA-текстура. Потолок 512 px + 256 кадров ограничивает
  худший случай; типичные веб-гифки сильно меньше.
- **256 кадров** — потолок `AnimatedTexture` движка. Хвост длинных гифок отбрасывается.
- **Минимальная задержка** кадра — 0.1 c при нулевой/слишком малой (как делают браузеры).

## Проверка

Декодер сверён покадрово (точное совпадение RGBA) с ground-truth из Pillow на наборе:
полные кадры, частичные кадры с прозрачностью + disposal 2, disposal 3, чересстрочный кадр.
Тест разовый (фикстуры генерятся Pillow в `/tmp`), в репозиторий не закоммичен.

## Где в коде

| Место | Что делает |
|---|---|
| `GifDecoder.decode()` | байты GIF → массив кадров `{image, delay}` |
| `GifDecoder._lzw_decode()` | LZW-распаковка данных кадра в палитровые индексы |
| `GifDecoder._blit()` / `_clear_rect()` | composition кадра на холст, disposal 2 |
| `GifDecoder._interlaced_row()` | маппинг строк при чересстрочности |
| `ImageLoader._decode()` (ветка `gif`) | точка входа: hint==gif → `_decode_gif` |
| `ImageLoader._decode_gif()` | сборка `ImageTexture` / `AnimatedTexture` из кадров |
