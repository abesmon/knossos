# Безопасность — статус и философия

> **Статус на сейчас.** Исполнение недоверенного **кода** страницей решено на уровне
> архитектуры: единственный способ привезти поведение — Luau в sandboxed VM с ограниченным
> capability API `document` (raw GDScript со страницы не поддерживается вовсе). Открытой
> остаётся **декларативная** поверхность: страница или пир могут инстанцировать в клиенте
> произвольные классы Godot со свойствами (`VrwebBuilder` через `ClassDB`), а
> `VrwebContentPolicy` пока работает в режиме passthrough без запрещающих правил — поэтому
> через декларации всё ещё достижимы исполнение кода (`GDScript`-ресурс/`script`-свойство,
> GDExtension), чтение ФС через свойства-пути и сетевые узлы. Это осознанно принятый риск
> стадии прототипа, и он должен быть явно известен каждому, кто пользуется vrweb-стеком:
> ходите только по доверенным страницам и подключайте только доверенные внешние ресурсы.
> Закрытие — opt-in safe profile, приоритеты в [едином roadmap](roadmap.md#p0--content-policy-safe-profile).

Этот документ — единая точка по теме: карта поверхностей риска (сводит воедино врезки,
разбросанные по остальным докам), философия подхода и планируемая модель защиты.

## Философия: инструменты вместо жёстких запретов

Дилемма, вокруг которой строится подход:

- **Блокировать произвольные куски не хочется.** Если зажать платформу до фиксированного SDK,
  получится ещё один VRChat — где можно только то, что разрешили разработчики. Это убивает
  главную идею открытого стека.
- **Исполнять код и инстанцировать классы с непроверенных сайтов — плохая затея.** Это понятно
  и не оспаривается.

Выбранное разрешение дилеммы: **не заниматься жёсткой безопасностью «за пользователя», а дать
пользователю инструменты обезопасить себя самому** — и пусть он решает, на что готов. Как в
браузерах решают про JS/куки/камеру per-site, а не «запретить всем навсегда».

Тот же принцип распространяется и за пределы исполнения кода — на **доверие к идентичности**.
Стек намеренно не защищает от выдачи себя за другого на уровне транспорта (самозаявленный
`user_id` подделываем — [network/ranks.md](network/ranks.md)); вместо этого он даёт материал для
проверки (подтверждённые сертификаты `nick@domain`), а решение, доверять ли неподтверждённым
пирам, оставляет **политикой клиента на усмотрение пользователя.** Везде одно и то же: транспорт
даёт возможности и факты, политику выбирает клиент/пользователь.

**Производительность — не вопрос безопасности.** VRWeb проектирует контракт взаимодействия, а
не оценивает эффективность authored content: ресурсоёмкая сцена, «тяжёлый» скрипт или частый
сетевой поток сами по себе не являются угрозой и не основание для запретов
([авторская свобода](README.md#авторская-свобода-и-граница-безопасности)). Watchdog, дедлайны и
бюджеты клиента ниже — это **containment аварии или намеренной атаки** (клиент обязан пережить
зависание и восстановиться), а не суждение о качестве кода; их числа — локальная policy
реализации, не норма стандарта.

## Решённое: page/item scripting

Единственный переносимый способ привезти поведение — Luau
([space/scripting.md](space/scripting.md)). Что это закрывает:

- каждая страница (и каждый экземпляр предмета `vrweb-item`) исполняется в собственной
  sandboxed VM: без Godot objects, ОС, файловой системы, произвольной сети и загрузки нативных
  библиотек;
- скрипт видит только `document` — versioned capabilities с opaque handles; результат каждого
  вызова подчинён [единому контракту](space/scripting-api.md#контракт-host-вызовов), а сеть
  ограничена `document.assets` (GET, схемные ограничения, лимиты);
- выбор файла пользователем (`document.files.pick`) — явное согласие через системный диалог;
  путь ОС скрипту не сообщается, байты идут в контент-адресуемый стор;
- страница не поставляет bytecode: клиент получает и интерпретирует исходник; opt-in SRI
  проверяет соответствие байтов ожиданию автора (но не «безопасность» логики);
- старого trust store/preflight для raw-кода нет, поскольку raw GDScript со страницы больше не
  поддерживается.

Стандартный capability pool безопасен сам по себе и не требует per-site permission prompts для
core API вроде locomotion: capability — это граница и обнаружение возможности, а не
делегирование safety пользователю. Instance ACL и код автора мира задают игровую семантику
поверх этой границы, но не расширяют доступ к engine/ОС.

## Карта поверхностей риска

| Поверхность | Риск | Статус | Где подробно |
|---|---|---|---|
| **Luau page/item scripts** | Попытка выйти за границу sandbox или злоупотребить выданным capability API | **Изолирован:** одна VM на realm, opaque handles, staged commit; deadlines/квоты клиента — containment аварий (не оценка эффективности). Hard allocator ceiling — открытый hardening | [space/scripting.md](space/scripting.md), [roadmap](roadmap.md#p0--scripting-transport-boundaries) |
| **Linked scripts transport** | Подмена/downgrade исходника по пути: redirect policy и hostile fixtures не завершены | **Открытый hardening** (P0): источник — только URL/hash из документа, байты от пира источником не являются | [roadmap](roadmap.md#p0--scripting-transport-boundaries) |
| **Page remote calls** | Пир вызывает локальный endpoint; изменённый клиент может вызвать любой опубликованный endpoint, не соблюдая проверки на своей стороне | **Изолирован:** transport закрепляет caller/target, script/endpoint/version, wire limits; handler автора решает allow/deny и может использовать только безопасный `document` API | [space/scripting-api.md](space/scripting-api.md#адресованные-remote-calls) |
| **VRWML страницы (декларации)** | `VrwebBuilder` инстанцирует через `ClassDB` любой класс Godot и ставит любые свойства из атрибутов недоверенной страницы: исполнение кода (`GDScript`-ресурс/`script`, GDExtension), свойства-пути → чтение ФС, сетевые узлы (SSRF/DoS) | **Принят** — `VrwebContentPolicy` пока passthrough, safe profile в P0 | [space/content-policy.md](space/content-policy.md), [roadmap](roadmap.md#p0--content-policy-safe-profile) |
| **Эфемерные действия пиров** | `vrweb-node`/`vrweb-patch` из действий пиров строятся тем же `VrwebBuilder.build_element` — тот же ClassDB-риск без захода на злонамеренную страницу. `vrweb-item` дополнительно заставляет клиент скачать item-документ по URL из действия пира: его скрипты попадают в обычный sandbox, но его VRWML идёт тем же декларативным путём | **Принят** (единая policy для document и live peer, но правил пока нет) | [network/ephemeral-changes.md](network/ephemeral-changes.md), [space/content-policy.md](space/content-policy.md) |
| **Флаш персистенции** | Запекает узлы пиров в страницу на сервере — после чего они грузятся и инстанцируются как обычные узлы страницы | **Принят** (транзитивно от двух предыдущих); правила стоит повторять и на серверной границе flush | [network/page-persistence.md](network/page-persistence.md) |
| **Runtime shaders** | `document.shaders.compile` передаёт недоверенный shader source компилятору GPU-драйвера: боевая поверхность драйверных багов/крашей | **Принят** как поверхность: структурные проверки формата и policy hook есть, содержимое shader не анализируется | [space/scripting-api.md](space/scripting-api.md#runtime-shaders-и-материалы) |
| **Внешний аватар** | Внешний `.tscn` со скриптами/произвольными классами отклоняется. HTTP принимает только data-only `.vrwml` через avatar allowlist/budgets | **Закрыт для `.tscn`**; VRWML ограничен `AvatarVrwmlPolicy` (deny script/source code, allowlist классов, бюджеты) | [client/avatars.md](client/avatars.md) → «Известные риски» |
| **`params` от пиров** | Словарь применяется как есть; влияет только на анимацию, но типы/диапазоны не валидируются | **Принят**, кандидат на валидацию при росте набора | [client/avatars.md](client/avatars.md) → «Известные риски» |
| **Выдача себя за другого (самозаявленный `user_id`)** | `user_id` не подписан: пир может скопировать чужой и унаследовать его ранг/авторство эфемерных объектов | **By design** — стек намеренно не защищает от этого на транспорте; подтверждённая идентичность (`nick@domain`) существует параллельно, а фильтрация недоверенных — планируемая **политика клиента** на усмотрение пользователя, не запрет стека | [network/ranks.md](network/ranks.md), [network/home-server.md](network/home-server.md) |
| **Кража ассета аватара** | Скачал байты → перезалил → «сам себе разрешил». «Аналоговая дыра»: клиент обязан получить байты, чтобы отрисовать | **Непредотвратимо** на открытом клиенте; манифест делает ворованное *нелегитимным*, а не невидимым | [client/avatars.md](client/avatars.md) → «Защита владения» |
| **Скачанные медиа (картинки/GIF/видео)** | Парсинг недоверенных байтов встроенными и своими декодерами | Частично смягчён (проверка сигнатур, свой GIF-декодер на GDScript, пре-скан JPEG) | [client/gif-support.md](client/gif-support.md), [client/implementation.md](client/implementation.md) |
| **Чат** | Флуд/оверсайз сообщений | Валидируется (`MAX_CHAT_CHARS`); эфемерный слой лимитирован (`MAX_OBJECTS`/`MAX_TTL`/`MAX_PROPS_BYTES`) | [network/multiplayer.md](network/multiplayer.md), [network/ephemeral-changes.md](network/ephemeral-changes.md) |
| **Идентичность (dev-режим)** | `--insecure-identity` / `VRWEB_INSECURE_IDENTITY=1` отключает строгую проверку источника ключей | Только для локальной разработки; UI показывает предупреждение | [network/home-server.md](network/home-server.md) |

## Что уже есть (существующие механизмы)

Точечные механизмы, работающие уже сейчас — они закрывают свои узкие места, но **не** отменяют
принятый декларативный риск сверху:

- **Luau sandbox и capability contract** — единственный путь исполнения кода страницы/предмета
  (см. «Решённое» выше).
- **Сертификаты идентичности** — криптографическое подтверждение `nick@domain` (RSA-SHA256,
  строгая проверка источника ключей) — [network/home-server.md](network/home-server.md).
- **Единая точка content policy** — document и live-peer декларации проходят через один
  `VrwebContentPolicy` (пока audit/passthrough): будущие правила нельзя будет обойти переходом
  со страницы на действия пира — [space/content-policy.md](space/content-policy.md).
- **`AvatarVrwmlPolicy`** — реальный работающий allowlist для внешних аватаров: deny
  script/source code, allowlist классов и ресурсов, бюджеты.
- **Манифест прав аватара** (AASA-style, host == trust) — легитимность ношения аватара;
  адрес манифеста выводится клиентом, не принимается от пира —
  [client/avatars.md](client/avatars.md).
- **Лимиты эфемерного слоя и чата** — потолки на число объектов, TTL, размер props, длину
  сообщений.
- **Правки только своих объектов** в эфемерном слое + валидация действий авторитетом —
  фундамент под систему прав ([network/ephemeral-changes.md](network/ephemeral-changes.md),
  [network/ranks.md](network/ranks.md)).

> **Не путать:** ключ запуска `--sandbox=<id>` — это **dev-изоляция** `user://` для запуска
> нескольких инстансов на одной машине ([network/multiplayer.md](network/multiplayer.md)),
> а НЕ security-песочница. Page scripting использует собственную Luau VM; этот ключ к ней не относится.

## Рекомендации пользователям

- Скрипты страниц изолированы, но **декларативная часть — нет**: пока safe profile не включён,
  заход на страницу означает доверие её разметке, а вход в комнату — доверие декларациям её
  участников (включая предметы, которые они приносят).
- Integrity проверяется только когда автор документа его указал; наличие SRI не доказывает
  безопасность логики скрипта и не защищает декларативную поверхность VRWML.
- Внешний `.tscn` отклоняется; для внешних аватаров используйте только `.vrwml` и всё равно
  проверяйте origin и внешние ресурсы документа.
- Не используйте `--insecure-identity` вне локальной разработки.
