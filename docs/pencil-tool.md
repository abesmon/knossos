# Карандаш и ластик (инструменты рисования)

> Инструмент поверх [эфемерного слоя](ephemeral-changes.md): рисование строит штрихи
> (`kind="stroke"`), которые ходят по сети как обычные эфемерные объекты. Решение **многослойное**
> и **engine-agnostic** по образцу `SceneChanges`: данные и геометрия штриха не зависят от движка,
> Godot-специфика — только в материализации и в слое инструмента.

## Что это

- Кнопка **2** достаёт **карандаш** (визуал появляется «в руке»). Зажатая **ЛКМ** рисует штрих в
  воздухе перед камерой.
- Повторное нажатие **2** меняет карандаш на **ластик** — он стирает свои штрихи, с которыми
  пересекается. Ещё раз **2** — инструмент убран (ЛКМ снова взаимодействует с порталами).
  Цикл: `НЕТ → КАРАНДАШ → ЛАСТИК → НЕТ`.
- Один непрерывный штрих (нажатие → ведение → отпускание ЛКМ) = **один** эфемерный объект.
  Рендерится **одним** мешем (труба вдоль полилинии) — цель «минимум мешей»: один draw-call на
  штрих, без физ-коллайдеров.

## Слои

| Слой | Файл | Роль |
|---|---|---|
| **Данные/геометрия** (чистая, агностичная, тестируемая) | [scripts/ephemeral/stroke_path.gd](../scripts/ephemeral/stroke_path.gd) `StrokePath` | накопление сэмплов, прореживание, упрощение (Douglas–Peucker), сериализация в плоский `props`; хит-тест полилинии. **Без нод/сети** — только точки и массивы float |

| **Материализация** | [actors/stroke/stroke.gd](../actors/stroke/stroke.gd) `StrokeActor` + ветка в [ephemeral_view.gd](../scripts/ephemeral_view.gd) | строит **один** меш по `props`; в группе `ephemeral_stroke` для хит-теста ластиком |
| **Инструмент** (Godot) | [actors/tools/tool_controller.gd](../actors/tools/tool_controller.gd) `ToolController` | состояние NONE/PENCIL/ERASER, визуал «в руке», сэмплинг/стирание; создаётся `Player` |

`StrokePath` тестируется в отрыве от движка — то же доказательство отвязки контракта от Godot, что и
у `SceneChanges`.

## Данные штриха

`props`: `{ "points": [x0,y0,z0,…], "color": [r,g,b], "width": <м> }`. Точки — **мировые**
координаты (`parent=""`), поэтому `StrokeActor` стоит в начале координат, а вершины абсолютны.
Полный контракт объекта — в [ephemeral-changes.md](ephemeral-changes.md#штрихи-второй-инструмент).

`StrokePath`:
- `add_sample(p)` добавляет точку, только если сдвиг ≥ `MIN_SAMPLE_DIST` (прореживание дрожания).
- `simplify()` — Douglas–Peucker (`SIMPLIFY_EPS`): выкидывает почти-коллинеарные точки → меньше
  вершин меша и байт в сети.
- `build_props(color, width)` — готовый `props`; `MAX_POINTS` и `MAX_PROPS_BYTES` ограничивают длину.
- статик `distance_to_polyline(flat, p)` — хит-тест ластика (минимум по отрезкам).

## Поток рисования (финализация при отпускании)

1. **press** (карандаш): `ToolController` заводит `StrokePath` и узел-превью (`StrokeActor` в режиме
   превью) в корне мира — рисуется сразу, локально, без сети (нулевая задержка для автора).
2. **ведение** (`_physics_process`, пока ЛКМ зажата): точка рисования = мировая позиция
   **маркера-остриё** инструмента (`_tip`, вынесен вперёд от «кисти» на `TIP_REACH`) — визуал
   остриё и фактический источник линии совмещены по построению. `add_sample` прореживает,
   `append_point` достраивает живой меш.
3. **release**: `simplify()`; если рисуемо (≥2 точек):
   - **в комнате**: один `request_scene_action({op:add, kind:"stroke", ttl:0, props})`. Превью
     держим, пока не придёт `scene_object_added` с **нашим** id (канонический `StrokeActor` от
     вьюхи) — затем снимаем превью. Зазор одной RTT закрыт без мигания.
   - **офлайн**: `add` не закоммитится (`has_authority()` ложно) — превью остаётся постоянным
     локальным штрихом (упрощённый меш + автор для ластика).

Никаких `update` по ходу: один штрих — ровно один объект, минимум трафика.

## Ластик

Пока ЛКМ зажата, для каждого узла группы `ephemeral_stroke`, чей `author` — наш (или локальный
офлайн-штрих), проверяем `hit_by(точка, ERASER_RADIUS)`:
- сетевой (есть `object_id`) → `request_scene_action({op:"remove", id})` (авторитет снимет,
  идемпотентно);
- локальный → `queue_free()` напрямую.

Чужие штрихи отсеиваются заранее — права владения (`SceneChanges`) бесполезного трафика бы не
пропустили, но и слать его незачем.

## Минимум мешей

Один `StrokeActor` = один `MeshInstance3D` с одним `ImmediateMesh` (труба из `SIDES` граней,
параллельный перенос рамки без закрутки), `StandardMaterial3D` unshaded. Хит-тест — чистая
математика по полилинии, **без** физ-коллайдеров на сегментах. Перестройка меша только при
ресинке/снимке (при финализации-на-отпускании `update` практически не приходит).

## Ввод

`Player` остаётся хабом ввода. Кнопка **2** → `ToolController.cycle()` (+ сигнал `tool_changed` →
подсказка в статус-строке). ЛКМ: при активном инструменте (`is_armed()`) press/release уходят в
инструмент, иначе — прежний путь `interact_at` (порталы/панели). Непрерывная выборка — в
`ToolController._physics_process`.

## Точки расширения

- **Визуал руки**: сейчас процедурный placeholder (цилиндр-карандаш / брусок-ластик) под камерой;
  заменяется на 3D-модель/анимацию без смены протокола.
- **Цвет/толщина**: цвет — стабильный оттенок по `user_id`; UI выбора цвета/толщины ляжет в
  `ToolController` и `props`.
- **Частичное стирание**: разрезать штрих на два (remove + 2×add либо update полного массива) —
  заметно сложнее, отложено.
- **Граффити-права**: стирать/править чужие штрихи — политика по `kind` в `SceneChanges`
  (capabilities), как предусмотрено в [ephemeral-changes.md](ephemeral-changes.md#права-владение--фундамент-под-систему-прав).
- **Рисование по поверхности**: сейчас точки только в воздухе на фиксированном расстоянии; режим
  «спрей по стене» (луч из камеры) — отдельная опция выбора точки в `ToolController`.
- **Персистенция**: `ttl=0` делает штрихи кандидатами на выгрузку — `points` ложатся в узел
  HTML-дерева так же, как прочие эфемерные объекты.
