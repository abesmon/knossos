# Голосовой чат

> Голос поверх существующего p2p WebRTC mesh из [multiplayer.md](multiplayer.md). Этап 1 —
> **полностью на GDScript, без зависимостей**: сырой PCM16 через data-канал. Намеренно
> неэффективно ради простоты; кодек спрятан за интерфейс, чтобы позже заменить на Opus, не
> трогая транспорт и захват.

## Главное ограничение

WebRTC в Godot (и нативный аддон `webrtc-native`, и web-обёртка) даёт **только data-каналы,
без медиа-треков**. Поэтому «правильный» WebRTC-аудио (SRTP, congestion control, браузерный
эхоподавитель) недоступен — голос передаётся как обычные данные поверх mesh, тем же приёмом,
что позиции и чат.

## Тракт

```
[A] микрофон → шина VoiceCapture (AudioEffectCapture)
      → даунмикс в моно (+gain)
      → VAD (детектор речи, гистерезис)
      → ресемпл 44.1/48 кГц → 24 кГц (LinearResampler)
      → нарезка на кадры по 40 мс
      → VoiceCodec.encode (PCM16)
      → NetworkManager.send_voice ──unreliable_ordered, канал 1──▶
[B] NetworkManager.voice_received → RemotePlayersView → RemotePlayer.push_voice
      → VoicePlayback: VoiceCodec.decode → AudioStreamGenerator (джиттер-буфер ~200 мс)
      → AudioStreamPlayer3D из позиции капсулы (пространственный звук)
```

## Компоненты

| Узел | Файл | Роль |
|---|---|---|
| `VoiceManager` (autoload) | [scripts/voice/voice_manager.gd](../scripts/voice/voice_manager.gd) | захват микрофона, VAD, ресемпл, кодирование, отправка; локальный сигнал «говорю» |
| `VoiceCodec` | [scripts/voice/voice_codec.gd](../scripts/voice/voice_codec.gd) | представление «на проводе» — сейчас PCM16 @ 24 кГц. Граница для замены на Opus |
| `LinearResampler` | [scripts/voice/linear_resampler.gd](../scripts/voice/linear_resampler.gd) | потоковый линейный ресемплер с переносом фазы между буферами |
| `VoicePlayback` | [scripts/voice/voice_playback.gd](../scripts/voice/voice_playback.gd) | `AudioStreamPlayer3D` + генератор + декодер на каждый поток пира |

Транспорт — в `NetworkManager` (`send_voice`/`_recv_voice`/`voice_received`), отдельный
**канал 1** (см. [multiplayer.md](multiplayer.md) → RPC). Захват включается только когда
**онлайн И голос включён в настройках И в комнате есть пир** — иначе микрофон отпускается.

## Ключевые решения

- **VAD с гистерезисом** (`OPEN_RMS`/`CLOSE_RMS` + `HOLD_SEC`): два порога и удержание не
  дают дребезга на паузах между словами и не режут тихие хвосты фраз. Молчащие пиры ничего
  не шлют — это главная экономия на mesh.
- **Канал 1, `unreliable_ordered`**: голос не блокирует состояние/чат (канал 0), а
  ретрансмиты не ломают реалтайм.
- **Джиттер-буфер = буфер `AudioStreamGenerator`** (~200 мс). Простой и устойчивый; при
  переполнении лишние семплы отбрасываем, удерживая задержку.
- **Пространственный звук**: голос играет из `AudioStreamPlayer3D` на капсуле говорящего —
  слышно, кто где. Неймплейт подсвечивается, пока пир говорит.
- **Кодек/декодер — инстансы** (а не статика): у Opus per-stream-состояние (FEC/PLC),
  поэтому энкодер один в `VoiceManager`, декодер — свой на каждую капсулу.

## Эффективность (почему это «этап 1»)

Сырой PCM тяжёлый. На один поток PCM16/моно: 24000 × 2 = **48 КБ/с ≈ 384 кбит/с**. В mesh
каждый шлёт в N−1 пиров; аплинк говорящего:

| Пиров | PCM16 24 кГц (сейчас) | Opus 24 кбит/с (цель) |
|---|---|---|
| 2 | 384 кбит/с | 24 кбит/с |
| 4 | 1.1 Мбит/с | 72 кбит/с |
| 8 | 2.7 Мбит/с | 168 кбит/с |

Плюс у PCM нет PLC (потерянный пакет = щелчок) и шумодава/AEC, а декод/микс N потоков в
GDScript упирается в CPU примерно на 4–6 одновременных голосах. VAD спасает: одновременно
говорят обычно 1–2 человека. Для «схода» на 2–4 человека этого этапа достаточно.

## Дальше (улучшение эффективности)

1. **Opus через узкий GDExtension** — заменить тело `VoiceCodec.encode/decode` и константы;
   транспорт, захват и воспроизведение не меняются. Даёт ~10× по трафику, FEC/PLC, VBR.
2. **Адаптивный джиттер-буфер** с номером последовательности в пакете (вместо «голого»
   буфера генератора) — короче задержка при ровной сети, устойчивее на рваной.
3. **Шумоподавление/AEC** (RNNoise/WebRTC APM через GDExtension) — иначе на колонках возможен
   возврат эха.
4. **SFU** — снять с mesh квадратичный аплинк на больших комнатах (далёкий этап, см.
   [vrweb-overview.md](vrweb-overview.md)).

## Предусловия и платформы

- В `project.godot` включён `audio/driver/enable_input=true` и автолоад `VoiceManager`.
- Нужен нативный `webrtc-native` (как и для всего мультиплеера, см.
  [multiplayer.md](multiplayer.md)); голос идёт по тому же mesh.
- На мобильных/web потребуется рантайм-разрешение на микрофон. Web-экспорт остаётся на
  PCM/Opus-over-datachannel — браузерный media-track недоступен без JS-интеропа.

## Проверка

1. Поднять сигналинг и поставить `webrtc-native` (см. [multiplayer.md](multiplayer.md)).
2. Запустить два инстанса (для изоляции `user://` — `--sandbox=A`/`--sandbox=B`), в обоих:
   настройки → «Онлайн» + «Голосовой чат (микрофон)» → один и тот же URL.
3. Говорить в одном инстансе — во втором слышен пространственный голос из капсулы, её
   неймплейт подсвечивается зелёным. В тишине поток не идёт (VAD).
