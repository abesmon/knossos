# Настройки

> Глобальные настройки приложения: автолоад `Settings` (`scripts/settings.gd`) хранит значения и
> персистит их в `user://settings.cfg`; экран `scenes/settings.tscn` (`scenes/settings.gd`) их
> редактирует. Подписчики (`NetworkManager`, `main`, `VoiceManager`) реагируют на сигнал `changed`.

## Автолоад `Settings`

`scripts/settings.gd` — единый источник правды для пользовательских настроек. Значения загружаются в
`load_settings()` при старте и пишутся обратно в `save()` по кнопке «Сохранить» (которая ещё применяет
аудио и эмитит `changed`). Пути учитывают dev-песочницу — см. [[sandbox-user-paths]] и
`Sandbox.resolve()`.

Файл `user://settings.cfg` — обычный `ConfigFile`, секции:

- `net` — `online_enabled`, `voice_enabled`, `signaling_url`, `nick`
- `browser` — `home_page` (домашняя страница, см. ниже)
- `graphics` — `fov` (угол обзора камеры игрока, 30–120°, по умолчанию 75)
- `voice` — `input_device`, `mic_gain`, `vad_threshold`
- `audio` — `output_device`, `vol_<bus>` для каждой шины (см. `docs/audio.md`)
- `identity` — `user_id` (постоянный самозаявленный id, см. `docs/ranks.md`)
- `avatar` — `uri` (адрес аватара, см. `docs/avatars.md`)

## Экран настроек (вкладки)

`scenes/settings.tscn` — overlay поверх UI, который `main` инстансит скрытым и показывает по «⚙» / Esc.
`TabContainer` с вкладками. Порядок задаётся в `_setup_tabs()`: «Мир» и «Пользователи» в сцене
объявлены последними (чтобы не ломать редакторские `tab_N/title`), а в рантайме переставляются на
позиции 2 и 3 через `move_child`; заголовки выставляются явно по имени узла.

0. **Основные** — домашняя страница, обзор камеры (FOV).
1. **Мир** — инфо о текущем инстансе; вкладка скрыта вне мира (см. ниже).
2. **Пользователи** — список пиров и ранги; вкладка скрыта вне онлайн-инстанса
   (`_update_users_availability`, см. `docs/ranks.md`).
3. **Сеть** — онлайн-режим, адрес сигнального сервера, ник, лицо, аватар, сетевой ID.
4. **Звук** — голосовой чат, микрофон, усиление/порог, вывод, громкости шин.
5. **Прочее** — управление кэшем.

## Вкладка «Мир»

Инфо о текущем инстансе. Доступна только когда мы в мире (загружена страница, `_current_url != ""`) —
по аналогии с «Пользователями» (`_update_world_availability`). Данные приходят от `main` при открытии
настроек: `_settings_overlay.open(_current_url, _page_meta)`.

`_page_meta` — «паспорт» страницы, который `main._extract_page_meta()` собирает из `<head>` в
`_on_fetched`:

- `title` — `<title>` или `og:title`;
- `description` — `description` / `og:description` / `twitter:description`;
- `thumbnail` — `og:image` / `twitter:image` (резолвится относительно базы страницы);
- `metas` — только человекочитаемые теги из белого списка `MEANINGFUL_META` (автор, сайт,
  ключевые слова и т.п.) с готовой подписью. Технические теги (`viewport`, `charset`,
  `theme-color`…) отфильтрованы — блок «О странице» показывает лишь осмысленное.

Превью грузит `ImageLoader` (тот же, что строит мир) — экземпляр живёт в самом экране настроек.

Кнопки:

- **Сделать инстанс домашним** — пишет `Settings.home_page = _current_url` и сразу сохраняет (как
  очистка кэша — не дожидаясь «Сохранить»); кнопка гаснет, если инстанс уже домашний.
- **Вернуться домой** (внизу) — эмитит `home_requested`; `main._on_home_requested` грузит
  `Settings.home_page`, а при пустом поле — **персональное пространство** домашнего сервера
  (запрашивает его адрес у сервера в момент нажатия, см. [personal-spaces.md](personal-spaces.md)).
  Доступна, если задана домашняя страница ИЛИ пользователь залогинен на сервере с
  `personal-spaces.v1`; иначе погашена («домашний инстанс не задан»).

## Домашняя страница

`Settings.home_page` (секция `browser/home_page`) — адрес, который загружается автоматически при
запуске. Задаётся на вкладке «Основные настройки».

> **Не путать с персональным пространством.** Домашняя страница — это произвольная закладка
> старта (может быть любой URL). Персональное пространство — хостируемый домашним сервером
> дом пользователя, вход в него отдельной кнопкой «Моё пространство» на вкладке «Аккаунт»,
> **независимо** от домашней страницы. Различие подробно — в [personal-spaces.md](personal-spaces.md).

- В `main._ready()` мышь при запуске всегда отпускается (UI становится фокусируемым); в режим
  перемещения пользователь входит сам кликом по 3D.
- Если `home_page` непустой — он подставляется в адресную строку и сразу грузится через
  `_navigate(home, "", true)` (как абсолютный ввод в омнибоксе, без базы; запись становится корнем
  истории навигации). Курсор при этом НЕ захватывается.
- Если поле пустое — фокус ставится в адресную строку, мир ждёт ввода адреса.

Пустое поле на сохранении остаётся пустым (без автозагрузки) — в отличие от поля сигнального сервера,
которое при очистке возвращается к дефолту.

## Обзор камеры (FOV)

`Settings.fov` (секция `graphics/fov`, 30–120°, по умолчанию 75 — дефолт `Camera3D` в Godot) —
вертикальный угол обзора камеры игрока. Ползунок на вкладке «Основные» применяет значение сразу
(`scenes/settings.gd._on_fov_changed` пишет в `Settings.fov` и эмитит `changed`, не дожидаясь кнопки
«Сохранить» — как громкости шин). `Player` (`actors/player/player.gd`) слушает `Settings.changed` и
выставляет `_camera.fov` в `_apply_fov()`; та же логика зовётся из `_ready()`. Зеркала
(`scripts/vrweb_mirror.gd`) копируют FOV активной камеры каждый кадр — правки распространяются на них
без отдельного кода.
