# Глоссарий

> Здесь собраны термины, которые в проекте означают **не то, что можно подумать**: либо они
> придуманы здесь и не встречаются в литературе, либо привычное слово используется в узком/ином
> смысле, либо одно слово имеет два разных значения в разных слоях. Однозначные вещи (сервер —
> это сервер) сюда не выносятся.

**Авторитет (комнаты)** — детерминированно выбранный пир, который *ровно один* выполняет
некоторое действие за всю комнату (валидирует эфемерные события, шлёт heartbeat видео). Не
«админ» и не сервер — просто роль, переходящая по правилу «первому id».
→ [network/authority.md](network/authority.md)

**Аппликатор** — узел аватара, который читает шину параметров и «применяет» её к чему-то
(копирует в `AnimationTree`, крутит кость, масштабирует рот). Модель VRChat: корень аватара
логики не имеет, всё делают аппликаторы. → [client/avatars.md](client/avatars.md)

**Бандл-пак** — аватары, поставляемые вместе с приложением (`res://avatars/avatar_N.tscn`),
адресуются как `vrwebavatar://N`. Противоположность внешним аватарам по URL.
→ [client/avatars.md](client/avatars.md)

**Документ** — то, что визуализирует Слой 1. Шире, чем «HTML-файл»: HTML + vrweb-теги +
досбор из эфемерного слоя = один документ на отрисовку; визуализации всё равно, откуда он.
→ [README.md](README.md#архитектура-слои-как-независимые-строительные-блоки)

**Домашний сервер** — сервер *идентичности* (аккаунт, сертификат `nick@domain`, друзья,
персональные пространства), аналог home server в Matrix. **Не путать** с «домашней
страницей»/«домашним инстансом» (`home_page` в настройках) — это просто стартовый URL клиента,
они никак не связаны. → [network/home-server.md](network/home-server.md),
[client/settings.md](client/settings.md)

**Инстанс** — полный «слепок» встречи: связка **URL ↔ 3D-пространство ↔ HTML-документ**, из
которого всё собирается (то есть Слой 1 и Слой 2 вместе). URL — его уникальный ключ, но инстанс
охватывает и документ, и пространство, а не только адрес. Приватный инстанс = тот же базовый URL
с уникальным query-параметром; планируется стандартизация `vrweb-instance=<код>`. Отдельных
«инстансов сверх URL», как в VRChat, нет и не планируется. **Не путать с сетевой комнатой** (ниже):
комната — лишь транспортная часть инстанса.
→ [network/multiplayer.md](network/multiplayer.md#комнатасеть--webrtc-room-инстанс--полная-связка)

**Карточка (идентичности)** — не UI-элемент, а reliable-RPC пакет `nick + PNG лица +
avatar_uri`, который пир шлёт при открытии p2p-канала. «Пришла карточка» = мы узнали, кто
перед нами. → [network/multiplayer.md](network/multiplayer.md)

**Комната** — слово с **двумя несвязанными значениями**:
- в space-доках — **3D-помещение**, порождённое секцией HTML (узел топологии);
- в network-доках — **WebRTC-room**: группа пиров, между которыми идут RPC/чат/голос. Она
  **ничего не знает о HTML-документе или 3D-пространстве** и завязана на URL только как на
  **непрозрачный уникальный идентификатор** для группировки. Это **не** то же, что инстанс:
  инстанс — полная связка (URL + документ + пространство), а сетевая комната — лишь транспортный
  слой поверх того же ключа.

Из контекста раздела обычно ясно, но при чтении поперёк слоёв — помнить.
→ [space/html-to-3d-topology.md](space/html-to-3d-topology.md),
[network/multiplayer.md](network/multiplayer.md#комнатасеть--webrtc-room-инстанс--полная-связка)

**Контракция** (устарело) — свёртка HTML-дерева снизу вверх в топологию: узлы сливаются, пока не наберут
вес комнаты; узел над ≥2 комнатами становится соединителем. Термин наш (не из литературы по
графам). → [space/html-to-3d-topology.md](space/html-to-3d-topology.md)

**Лицо** — текстура игрока 256×256 PNG, накладываемая на аватар через маркер-слот. Это не
весь аватар, а одна подменяемая текстура («аватарка» в бытовом смысле).
→ [client/avatars.md](client/avatars.md)

**Манифест (прав аватара)** — JSON-файл-сиблинг рядом с файлом аватара (`*.manifest.json`),
объявляющий, кто может его носить (AASA-style, доверие = контроль хоста). Не подпись и не DRM.
→ [client/avatars.md](client/avatars.md)

**Песочница / `--sandbox=<id>`** — **dev-изоляция** каталога `user://` для запуска нескольких
инстансов клиента на одной машине. НЕ security-песочница исполнения кода (её сейчас нет — см.
[security.md](security.md)). → [network/multiplayer.md](network/multiplayer.md)

**Призрак** — **два разных значения**: (1) *ghost-пир* — grace-период (`peer_ghosted`) при
переподключении, когда капсула не удаляется мгновенно; (2) *голосовой призрак* — концепт
voice-only пользователя, чьим аватаром в мире управляет автоматика.
→ [network/multiplayer.md](network/multiplayer.md#переподключения-и-защита-от-морганий),
[graceful-degradation.md](graceful-degradation.md)

**Пузырь** — временный портал «ушёл сюда» (30 с), который остаётся на месте пира, ушедшего на
другую страницу. Первый инструмент эфемерного слоя.
→ [network/ephemeral-changes.md](network/ephemeral-changes.md)

**Самозаявленный `user_id` (vs подтверждённая идентичность)** — случайный id, которым пир
представляется; его можно подделать. Вопреки первому впечатлению это **не временная дыра до
«настоящей» верификации**, а постоянная часть модели: стек намеренно не защищает от выдачи себя
за другого, а подтверждённая идентичность `nick@domain` от домашнего сервера живёт **параллельно**
как опциональный якорь. Доверять ли неподтверждённым — политика клиента, не гарантия транспорта.
→ [network/ranks.md](network/ranks.md), [network/home-server.md](network/home-server.md),
[security.md](security.md)

**Соединитель** () — узел топологии, связывающий ≥2 комнаты (порождается контракцией). В
геометрии становится хабом/развилкой, сам комнатой не является.
→ [space/html-to-3d-topology.md](space/html-to-3d-topology.md)

**Таймкипер** — частный случай авторитета: пир с наименьшим id, который шлёт heartbeat
синхронизации видео. → [network/authority.md](network/authority.md),
[client/video-player.md](client/video-player.md)

**Топология vs геометрия** — жёсткая граница пайплайна: *топология* — сериализуемый артефакт
`Space` **без единой 3D-координаты** (комнаты, связи, объекты); *геометрия* — 3D-мир,
детерминированно построенный из него по seed. Контракт между `TopologyBuilder` и
`WorldGenerator`. → [implementation.md](implementation.md)

**Флаш** — закрепление эфемерных изменений в самой странице: клиент отправляет наработки, сайт
запекает их в HTML (Слой 4). Контракт **конечного клиента и конечного сайта** — федерация,
домашние серверы и сигналинг тут ни при чём.
→ [network/page-persistence.md](network/page-persistence.md)

**Шина параметров** — локальное хранилище именованных сигналов аватара (`AvatarParameters`):
источник пишет (`LookPitch`, `Voice`…), аппликаторы читают. По сети уезжает её снимок-словарь.
→ [client/avatars.md](client/avatars.md)

**Эфемерный слой / эфемерные изменения** — объекты, добавленные в общий мир на лету (не из
исходного HTML): живут в реалтайм-сессии, по умолчанию исчезают; могут быть закреплены флашем.
→ [network/ephemeral-changes.md](network/ephemeral-changes.md)

**vrweb-теги** — кастомные теги в HTML, имена которых 1-в-1 совпадают с классами Godot
(`<VRWebVideoPlayer>`, `<ExtScene>`…); клиент строит из них сцену. Ранний неймспейс `vr-*` из
заметок не используется. → [space/vrweb-tags.md](space/vrweb-tags.md)

**VRWeb / Knossos** — стандарт (набор технологий) / его референсная реализация (клиент +
домашний сервер). → [README.md](README.md#два-имени-vrweb-и-knossos)
