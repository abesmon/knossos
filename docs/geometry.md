# Геометрия пространства (фаза F)

Геометрическая фаза из [html-to-3d-topology.md](html-to-3d-topology.md) §11. Потребляет
**только** сериализуемый артефакт топологии (комнаты/коннекторы/объекты/вложенность/метки) +
`seed` и сочиняет конкретное навигируемое 3D-пространство. Вся реализация —
[scripts/world_generator.gd](../scripts/world_generator.gd).

Документ описывает текущую модель раскладки **v3 «кварталы на сетке» + нерегулярные формы
комнат**. Раздел [«История итераций»](#история-итераций) фиксирует, что пробовали и почему
отвергли — чтобы не ходить по кругу.

> **Статус.** Форма комнат доведена до играбельного состояния (прямоугольники по умолчанию +
> редкая компактная нерегулярность; баги стен/коридоров/вывесок/панелей закрыты). **Не сделано:**
> использование интерьера больших комнат (центр пустой) — см. [«Дальше»](#дальше).

---

## 0. Конвейер (`_build`)

Геометрия строится не за один кадр, а порциями (тайм-слайс по `BUILD_BUDGET_MS`), чтобы тяжёлая
страница не вешала главный поток. Фазы:

| Фаза | Функция | Что делает |
|---|---|---|
| 1. Замер объектов | `_measure_objects` | футпринт каждой панели (ширина×высота, м) из типа/размеров/текста |
| 2. Размер + **форма** комнаты | `_compute_room_dims` → `_compute_room_dim` | площадь под объекты, снап к клеткам; **форма футпринта** (см. §2) |
| 3. Утряска раскладки | `_relax_layout` | пружины родитель↔ребёнок + расталкивание + притяжение к сетке + гравитация по сиду |
| 3.5. Снап на сетку | `_snap_to_cells` | жёсткий снап в клетки, кластерная упаковка коннектор+дети, зазоры-улицы |
| **2.5. Доводка формы** | `_compute_room_shapes` | форма коннекторов; откат кластер-детей в прямоугольник (см. §2.4) |
| 4. Позиции/occupancy | `_finalize_positions`, `_fill_occupancy` | центры комнат в мире; занятые клетки + карта bbox |
| 5. Роутинг коридоров | `_route_corridors` | ортогональные пути по улицам, проёмы в стенах |
| 6. Геометрия | `_build_room`, `_build_atmosphere` | полы/стены/объекты/вывески/коридоры/свет |

Случайность — только от `seed` (цвета/атмосфера/выбор формы/раскладка пентамино); при одном
сиде раскладка детерминирована.

---

## 1. Сетка

- `GRID = 3.0 м` — клетка = ширина коридора/проёма. Вся геометрия выровнена по целым клеткам,
  поэтому каждая стена ложится на линию сетки.
- `STREET_CELLS = 1` — зазор-улица между bbox комнат (по нему идут коридоры).
- `ROOM_MIN_CELLS = 2` — минимальная сторона комнаты.
- Комната хранится как: `_room_cells[id]` — **bbox** в клетках (ширина, глубина);
  `_room_shape[id]` — **множество клеток пола** (полимино внутри bbox); `_room_size[id]` — bbox в
  метрах; `_positions[id]` — центр bbox в мире.

---

## 2. Форма комнаты

**Главный принцип (выведен итерациями): реальные помещения — прямоугольники.** По умолчанию
комната — прямоугольник; нерегулярность редкая, **компактная** (без «кишок»-тоннелей) и
архитектурная.

### 2.1 Комнаты (`_make_room_shape`)

1. `base = _rect_dims(cw, id, ROOM_ASPECT_MAX)` — **неравносторонний** прямоугольник той же
   площади (аспект из сида, ориентация случайна). Не квадрат — так интереснее.
2. **Только комнаты ≥ 4×4** могут отклониться от прямоугольника, и то с вероятностью
   `IRREGULAR_CHANCE`. Мелкие (2×2/3×3) всегда прямоугольные.
3. Если отклоняется — один модификатор (все на базе прямоугольника, осевые стены, мин. ширина
   рукава 2 клетки):
   - **скос** (`_carve_octagon`) — срез четырёх углов «лесенкой» (октагон/фаска);
   - **угол** (`_carve_L`) — убран один угловой блок (L-форма);
   - **ниша** (`_carve_notch`) — прямоугольная выемка-альков из середины стороны (глубина < половины);
   - **крест** (`_carve_plus`) — плюс с рукавами; только для **небольших** комнат (`area < BIG_AREA_CELLS`);
   - **пентамино** (`_aggregate_pentominoes`) — «совсем странная» компактная клякса; тоже только мелкие.
4. **Страховка:** если срез выродил форму (`shape.size() < 3`) — откат к прямоугольнику. Без неё
   «скос» на 2×2 убирал все 4 угла = пустая комната без пола/стен (см. историю багов).

### 2.2 Пентамино (`_aggregate_pentominoes`)

Набирает `ceil(area/5)` пентамино (12 свободных фигур, случайный поворот) и **компактно** их
стягивает: каждый кусок приклеивается боком к фигуре, но из `PENTO_TRIES` случайных вариантов
берётся тот, что даёт наименьший bbox (`_placement_score` = площадь + штраф за вытянутость) —
это убирает тоннели. Закрытые дырки заполняются полом (`_fill_holes`, флуд-филл снаружи).

### 2.3 Коннекторы (`_compute_room_shapes`, фаза 2.5)

Коннекторы — узлы-развилки; их форму доводим **после** кластерной упаковки (их bbox растёт в
`_build_cluster_layout`). Только **мягкие** срезы (`квадрат/фаска/октагон` — без сильных
изгибов): `_pick_connector_shape`. Кластер-якоря (коннектор с приклеенными детьми) **не режем** —
детям нужна прямая стена.

### 2.4 Кластер-дети

Комната, приклеенная вплотную к коннектору (`_direct_child`), в фазе 2.5 **возвращается в
прямоугольник** — склейке нужна прямая стена.

### 2.5 Константы-ручки

```gdscript
SHAPE_ENABLE        # false — все комнаты прямоугольные (как раньше)
IRREGULAR_CHANCE    # 0.4 — доля «непрямоугольных» комнат (≥4×4)
ROOM_ASPECT_MAX     # 1.7 — вытянутость прямоугольной комнаты
BIG_AREA_CELLS      # 30 — комнаты ≥ этой площади только архитектурные (rect/L/ниша; без креста/пентамино)
PENTO_TRIES         # 6 — попыток размещения куска при компактной пентамино-сборке
CONNECTOR_ASPECT_MAX# 2.2 — вытянутость прямоугольника коннектора
ROOM_SHAPE_FORCE    # "" = по сиду; иначе форсить форму комнаты (для оценки): прямоугольник/скос/угол/ниша/крест/пентамино
SHAPE_FORCE         # форсить архетип коннектора
```

## 3. Стены, проёмы, двери (`_build_walls`)

Стены обходят **контур полимино**: для каждой клетки пола каждое ребро, смотрящее на не-пол
(наружу/в срез), становится сегментом стены длиной `GRID` (`_add_edge_wall`, сдвиг внутрь на
полтолщины ⇒ внешняя грань на линии сетки, стыки соседних комнат грань-в-грань без z-fight).

**Двери — с перемычкой (`_edge_wall`).** Ребро-проём (совпадает с `_room_openings`) режется
**только на высоту двери** `DOOR_HEIGHT = 2.4 м`; выше остаётся стена-перемычка. Это: (а) делает
проход похожим на дверь, а не дыру до потолка; (б) **даёт опору вывеске-заголовку** (см. §5).

---

## 4. Объекты на стенах (`_wall_spans`, `_place_objects`)

Объекты ставятся по **4 внешним стенам bbox** в порядке чтения (обход по часовой:
nz→px→pz→nx), слева направо внутри стены. Из стен вычитаются проёмы и **срезанные участки**
(`_carved_gaps` — где краевая клетка bbox не пол). Так как комнаты в основном прямоугольные,
внешних стен достаточно, и объекты стоят ровно, не залезая на утопленные/тонкие стены вырезов.

- `OBJECT_INSET = 0.35 м` — объект висит **на стене** (как картина/табличка), а не в воздухе
  перед ней. (Было 1.2 — панели читались как «таблички, висящие вдоль дорожек».)
- При переполнении (объектов больше, чем стен) позиция **клампится** в пределы стены — лёгкое
  перекрытие в конце вместо вылезания объекта наружу.
- Панели текста/картинок центрируются по `EYE_LEVEL`; высокие приподняты, чтобы низ не уходил в пол.

---

## 5. Вывеска-заголовок над дверью (`_build_room_title`)

Заголовок секции (первый объект-`heading` комнаты) уезжает **лентой-перемычкой над входным
проёмом** (вход = проём от родителя, `_room_entrance`), читается с обеих сторон. Садится в зону
перемычки `[DOOR_HEIGHT .. верх]` — там есть полотно стены, поэтому **не висит в воздухе**.

Плоскость стены проёма берётся из самого проёма (`op["off"]`, считается в `_add_opening` через
`_edge_off`), а **не из края bbox** — иначе на нерегулярной форме (или утопленной стене) вывеска
уезжала бы в пустоту.

---

## 6. Коридоры (`_route_corridors`)

BFS по клеткам-улицам от корня; сначала строго по свободным улицам, иначе с проходом по чужим
коридорам, иначе прямой манхэттен-фолбэк. Проёмы регистрируются у обеих комнат.

**Коридор не пересекает комнату.** `_fill_occupancy` помечает **весь bbox** каждой комнаты в
`_bbox_cells` (включая срезанные клетки), а `_passable` считает их непроходимыми. Без этого
коридор нырял в срез/нишу внутри bbox — «дорожка сквозь комнату». Помощники роутинга
(`_border_cells`, `_border_toward`) работают по реальным клеткам формы, а не по краям bbox.

---

## 7. Атмосфера

Свет/небо процедурно из данных страницы (`_build_atmosphere`): палитра из CSS-фонов комнат →
тон неба/солнца; «богатство» (число листьев корня) → высота солнца и яркость. Сид задаёт азимут.

## История итераций

Хронология решений этой сессии (что пробовали → к чему пришли):

1. **v1 — срез углов прямоугольника** (октагон/ромб/крест). Вывод: формы слишком регулярные
   (все симметричные), и квадрат `W×W` скучен.
2. **v2 — комнаты из пентамино + контур стен + объекты по всему контуру.** Вывод: **слишком**
   нерегулярно — почти все комнаты «кишки»-тоннели; стрингованные мелкие комнаты; большие — бесформенные
   кляксы. Раскладка объектов по всему контуру → каша и «вразнобой».
3. **v3 — прямоугольник по умолчанию + редкая компактная нерегулярность.** Вероятность
   `IRREGULAR_CHANCE`, модификаторы на базе прямоугольника, компактное пентамино (стягивание к
   центру), большие комнаты только архитектурные. Объекты вернули на **внешние стены bbox**
   (порядок, без каши). **Это текущая модель.**

**Закрытые баги (после перехода на формы):**

- *Коридор сквозь комнату* — `_bbox_cells` + `_passable` (§6).
- *Вывеска висит в воздухе* — две причины: (а) проём во всю высоту без полотна над дверью → перемычка
  `DOOR_HEIGHT` (§3, §5); (б) **вырожденная пустая форма** (скос 2×2 убирал все клетки) → гейт «≥4×4» +
  страховка `shape.size() < 3 → прямоугольник` (§2.1). Вторая причина была главной.
- *Объекты вылезают/вразнобой* — возврат на внешние стены bbox + клампинг переполнения (§4).
- *Таблички висят перед стеной* — `OBJECT_INSET` 1.2 → 0.35 (§4).

Воспроизведение и проверка шли на `https://ru.wikipedia.org/wiki/Все_вы_зомби` (28 комнат, много
навигационных меню). После фиксов: 0 вырожденных комнат, 0 меток далеко от стен.

## Дальше

**Рычаг №2 — интерьер больших комнат.** Объекты висят только на стенах ⇒ центр больших комнат
пустой. Задумка (не реализована): таксономия привязок объектов (настенный / напольный / на
поверхности) + **маршрутный хребет** через комнату от входа, вдоль которого объекты ставятся в
порядке чтения с гарантией прохода. Это же даёт осмысленную расстановку напольных объектов
(стенды/кафедры/порталы) в интерьере.

**Рычаг №3 — вертикаль.** Всё висит на `EYE_LEVEL`; верх/низ стен простаивает. Развеска в 2–3
ряда снизит потребную длину стены (и площадь комнаты).
