# VRWeb — обзор проекта

> **Суть:** открытый клиент (браузер) на Godot 4, который рендерит любую существующую HTML-страницу как навигируемое 3D-пространство. Никакой поддержки со стороны сайтов не требуется.

---

## Концепция

VRWeb — это интерпретатор: он берёт произвольный HTML-документ и превращает его в 3D-сцену, по которой можно ходить. Аналогия — обычный браузер интерпретирует HTML в 2D-страницу; VRWeb делает то же самое, но в трёх измерениях.

Ключевые принципы:
- **Любой сайт без изменений** — страница не обязана знать о VRWeb
- **Клиент решает всё** — вся логика трансляции на стороне клиента
- **Слои независимы** — каждый следующий этап (мультиплеер, кастомные теги, персистентность) добавляется поверх, не ломая предыдущий
- **Открытый стандарт** — любой может написать альтернативный клиент (плагин для браузера, мобильное приложение и т.д.)

---

## Архитектура: четыре слоя

### Слой 1 — Рендеринг HTML → 3D

Основной слой. Клиент получает HTML, парсит его и строит 3D-сцену по правилам маппинга.

**Режимы работы:**
- `hybrid` — HTML рендерится в 3D, кастомные теги дополняют сцену
- `override` — HTML игнорируется, используются только кастомные `vr-*` теги
- `full replace` — только кастомная разметка, HTML полностью заменён

### Слой 2 — Реалтайм (WebRTC)

Мультиплеер: люди на одной странице видят капсулы друг друга, слышат голос, видят чат. Эфемерные объекты (граффити и т.д.) живут в этом слое и по умолчанию не сохраняются.

### Слой 3 — Идентичность (федеративная)

Опциональный. Каждый пользователь регистрируется на своём home server'е, который хранит аватар и метаданные. Без регистрации — анонимная капсула. Верификация через публичный ключ (аналог ActivityPub / W3C DID).

### Слой 4 — Персистентные изменения мира

Клиент шлёт событие → сервер обрабатывает → обновляет страницу → клиент ре-рендерит. Авторизация опциональна. Стандарт для server-to-server договорённостей.

## Правила трансляции HTML → 3D (Слой 1)

### Структура → геометрия

| HTML-элемент | 3D-аналог |
|---|---|
| `<body>` | корневое пространство |
| `<header>` | фасад / вход |
| `<main>`, `<article>`, `<section>` | отдельные комнаты / зоны |
| `<nav>` | коридор или хаб с указателями |
| `<aside>` | боковое ответвление |
| `<footer>` | подвал (буквально) |

Для сайтов без семантики (div-soup) — визуальные эвристики по размеру, позиции и плотности контента блоков.

### Контент → объекты

| Контент | 3D-аналог |
|---|---|
| `<h1>`–`<h6>` | крупные объекты, масштаб по уровню |
| `<p>` | текстовые панели / таблички |
| `<img>` | текстуры / картины на стенах |
| `<video>` | экраны / телевизоры |
| `<a href="...">` (внутренняя) | дверь в другую комнату |
| `<a href="...">` (внешняя) | портал в другой мир |
| `<button>`, `<input>`, `<form>` | интерактивные панели |
| `<ul>`, `<ol>` | кластеры объектов |

### CSS → визуальные свойства

| CSS | 3D-аналог |
|---|---|
| `background-color` | цвет стен / пола |
| `font-size` | масштаб объекта |
| `display: none` | не рендерится |
| `z-index` | глубина в пространстве |

### Граф ссылок → топология

- Слабосвязанные части сайта → острова
- Частые переходы между страницами → мостики
- Страницы с высоким in-degree → большие центральные хабы

## Кастомные теги (Слой 1, расширение)

Теги с неймспейсом `vr-*` — обычный браузер их игнорирует, VRWeb их читает:

```html
<!-- Вставить 3D-сцену -->
<vr-scene src="world.glb"/>

<!-- Режим замены HTML -->
<html vr-mode="override">

<!-- Декларативное событие -->
<vr-object vr-on="player_enter" vr-action="animate"/>
```

Контент сцены — GLTF/GLB (открытый стандарт, Godot поддерживает нативно).

---

## Скриптинг внутри 3D-сцен

Два подхода, от безопасного к мощному:

1. **Декларативные события** (`vr-on`, `vr-action`) — безопасно, предсказуемо, реализуется в первую очередь
2. **WebAssembly** — любой язык, хорошая изоляция через sandbox, более долгосрочная цель

> GDScript инлайн в HTML — **не рассматривается**: исполнение произвольного кода с любого сайта создаёт неприемлемые риски безопасности.

## Мультиплеер (Слой 2)

- **Комната = URL страницы**
- **Транспорт:** WebRTC (голос, чат, позиции, эфемерные объекты)
- **Сигнальный сервер:** нужен только для handshake; после установки соединения — p2p напрямую
- **Федерация серверов:** сигнальные серверы пересылают сообщения друг другу (аналог Matrix); пользователи с разных серверов попадают в одну комнату
- **Масштабирование:** при 50+ людях в комнате mesh WebRTC не справляется — нужен SFU (Selective Forwarding Unit)

Простой вариант для прототипа: один сигнальный сервер на комнату, клиент выбирает через метатег на странице (или дефолтный публичный сервер).

## Технический стек

**Клиент:** Godot 4

**Ключевые проблемы реализации в Godot:**
- Парсинг HTML — через GDExtension + libxml2 или аналог (нативного HTML-парсера в Godot нет)
- JS-рендеринг SPA — в первой версии не поддерживается: ожидаем что HTML уже готов после загрузки
- WebRTC — официальный модуль `godot-webrtc`
- 3D-контент — GLTF/GLB нативно поддерживается Godot

**Получение HTML:**
- Прямой `HTTPRequest` (без исполнения JS)
- Позже: прокси или headless Chromium для JS-рендеринга

## Этапы разработки (Roadmap)

### Phase 1 — Singleplayer 3D Browser
**Цель:** ходить по любой странице в 3D, кликать по ссылкам для навигации

- [ ] HTTPRequest HTML из Godot
- [ ] Парсинг HTML → дерево узлов
- [ ] Маппинг: `section` → комната, `h1` → объект, `a` → портал/дверь
- [ ] Базовая навигация от первого / третьего лица
- [ ] CSS `background-color` → цвет окружения
- [ ] Кликабельные `<a>` — телепорт на новую страницу

**Deliverable:** можно ввести URL и погулять по странице в 3D.

---

### Phase 2 — Multiplayer
**Цель:** встречать людей на любой странице

- [ ] Сигнальный сервер (WebSocket)
- [ ] WebRTC handshake через Godot-модуль
- [ ] Капсулы других игроков в сцене
- [ ] Синхронизация позиций
- [ ] Голосовой чат
- [ ] Текстовый чат

**Deliverable:** несколько человек одновременно на одной странице видят друг друга.

---

### Phase 3 — Identity + Custom Worlds
**Цель:** аватары, персистентность, расширенная разметка

- [ ] `vr-*` теги — парсинг и рендеринг
- [ ] GLTF/GLB инлайн в HTML
- [ ] `vr-mode="override"` — полная замена
- [ ] Декларативные события (`vr-on`, `vr-action`)
- [ ] Home server — хранение аватара, pubkey-идентити
- [ ] Анонимный режим (капсула без аккаунта)
- [ ] Server-to-server протокол для мутаций мира

**Deliverable:** кастомные персистентные 3D-миры поверх обычных HTML-страниц.
