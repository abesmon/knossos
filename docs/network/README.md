# docs / network

Здесь собраны документы про **сетевой слой** VRWeb: как люди встречаются на одной странице,
как устроены права в комнате, как изменения мира живут и сохраняются, и как работает
федеративная идентичность через домашние сервера.

Соответствие слоям обзора ([../README.md](../README.md)): мультиплеер — Слой 2, домашний
сервер — Слой 3, персистентность — Слой 4.

## Реалтайм-комната (Слой 2)

- [multiplayer.md](multiplayer.md) — мультиплеер: WebRTC-mesh, сигнальный сервер только для
  handshake, синхронизация капсул, чат, переподключения.
- [authority.md](authority.md) — авторитет комнаты: кто из пиров считается источником правды
  для общего состояния сцены.
- [ranks.md](ranks.md) — ранги: таблица прав в комнате (кто что может менять).
- [ephemeral-changes.md](ephemeral-changes.md) — эфемерные изменения сцены (граффити и т.п.),
  живущие в реалтайм-слое и по умолчанию не сохраняемые.

## Идентичность и персистентность (Слои 3–4)

- [home-server.md](home-server.md) — домашний сервер: pubkey-идентичность, сертификаты
  `nick@domain`, discovery, федерация (референс — `homeserver/` на FastAPI).
- [personal-spaces.md](personal-spaces.md) — персональные пространства как модуль домашнего
  сервера.
- [presence.md](presence.md) — presence «где люди»: сводка занятых страниц от домашнего
  сервера, приватность и модель доверия.
- [page-persistence.md](page-persistence.md) — персистенция страницы: флаш эфемерного слоя в
  страницу, протокол сохранения изменений мира.
