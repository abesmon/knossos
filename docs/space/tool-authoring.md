# Как сделать свой инструмент (руководство автора)

> Практическое руководство поверх [portable-tools.md](portable-tools.md) (архитектура и
> принципы) и [scripting-api.md](scripting-api.md) (справочник API): как автор собирает
> **инструмент-предмет** — переносимый блок «VRWML + Luau», который добавляется в любой мир
> без обновления клиента. Референсы, с которых стоит начинать копированием:
> [pencil.html](../../test_pages/items/pencil.html) (непрерывное действие по прицелу),
> [eraser.html](../../test_pages/items/eraser.html) (модификация чужих артефактов слоя),
> [image_frame.html](../../test_pages/items/image_frame.html) (импорт файла пользователя),
> [color_cube.html](../../test_pages/items/color_cube.html) (минимальный предмет).

## Модель в одном абзаце

Инструмент — это **item-документ**: обычная маленькая страница с блоком `<vrwml>`, где лежит
[`<VRWebGrabbable>`](grabbable.md) (физический носитель — его берут в руку и видно другим;
возможность отобрать задаёт `theft`) и `<script type="application/vrweb+luau">` (поведение). В мир предмет
приходит объектом [эфемерного слоя](../network/ephemeral-changes.md#переносимые-предметы-kindvrweb-item)
`kind="vrweb-item"` с `props.src` — каждый клиент сам скачивает документ и исполняет его в
**собственном realm** со **стандартным** capability pool. Никакой регистрации, магазина или
разрешений: если API есть в стандарте — им можно пользоваться (модель браузера,
[security.md](../security.md)). Что инструмент делает в инстансе — политика владельцев
инстанса (ранги, права слоя), а не формата.

## Анатомия item-документа

```html
<!DOCTYPE html>
<html>
<head>
  <script type="application/vrweb+luau" id="my-tool.behavior">
    -- поведение: см. разделы ниже
  </script>
</head>
<body>
  <vrwml>
    <Resource id="Body" type="BoxMesh" size="Vector3(0.2,0.2,0.2)"/>
    <VRWebGrabbable id="my-tool" mode="adjustable" theft="deny">
      <MeshInstance3D mesh="SubResource:::Body"/>
    </VRWebGrabbable>
  </vrwml>
</body>
</html>
```

- **`<VRWebGrabbable id=...>`** — носитель. Атрибуты: `mode="fixed"` (снап в авторский
  `grip` — инструменты с осмысленной позой в руке: карандаш, «пистолет») или
  `mode="adjustable"` (естественный хват + подстройка колесом/СКМ — реквизит);
  `theft="allow|deny"` (`deny` рекомендуется для инструментов, чтобы их нельзя было
  отобрать во время действия). Полный контракт — [grabbable.md](grabbable.md).
- **Скрипт** получает стандартный `document` и адресует свои узлы по `id`
  (`document.query("#my-tool")`). Скриптов может быть несколько — они делят realm предмета.
- **Namespace автоматический**: id скриптов и grabbable-адреса каждого экземпляра предмета
  префиксуются id объекта слоя — два одинаковых предмета в комнате не конфликтуют ни в
  state, ни в hold-состоянии. Внутри документа пишите короткие локальные id.
- Ассеты — как у любой страницы: `<Resource>` инлайном, байты — `<VRWebBlob>`,
  внешние URL — через `document.assets`.

## Ввод: цикл жизни в руке

| Событие | Когда | Где эмитится |
|---|---|---|
| `grab {user_id, hand}` | предмет взяли | у всех клиентов |
| `use {user_id, hand}` | нажали основное действие с предметом в руке | только у держателя |
| `use_end {user_id, hand}` | отпустили основное действие | только у держателя |
| `drop {user_id, hand}` | предмет положили/отобрали/авто-release | у всех клиентов |

Три базовых паттерна ввода:

1. **Мгновенное действие** — вся логика в `use` (штамп, выстрел, звонок).
2. **Непрерывное ведение** — `use` ставит флаг, `document.on_update` каждый кадр читает
   `document.player.aim()`, `use_end` финализирует (карандаш). `aim()` всегда отдаёт
   `origin`/`direction` (действие «в воздух»), при попадании — `position`/`normal`/
   `distance`/`target` (html id узла под прицелом).
3. **Прицел + подтверждение** — `use` фиксирует точку прицела и продолжает асинхронно
   (рамка картинок: точка → `files.pick` → размещение по callback).

### Локальное превью прицеливания

Инструменту почти всегда нужен видимый прицел — луч и маркер места применения. Объявите их
в VRWML предмета как `top_level="true"` узлы (живут в мировых координатах) и ведите каждый
кадр из `document.on_update`, ставя `global_transform`. Позу на поверхности даёт
`document.values.transform_facing(position + normal * gap, normal, direction)` — она же
корректно обрабатывает пол и потолок; в пустоте разворачивайте маркер лицом к игроку
(`transform_facing(point, -direction, direction)`).

**Превью локально по построению**: сцена предмета строится у всех, но показывать превью
должен только клиент держателя — сравните `event.user_id` из `grab` с
`document.players.local_info().user_id` и гасите узлы на `drop`. Так другие игроки видят
предмет в руке, но не ваш прицел.

**Всегда обрабатывайте `drop`**: держатель может положить предмет (G) или его могут отобрать
посреди действия — незавершённое состояние (флаги, превью) нужно откатить. Оно же —
страховка от «залипшего» инструмента.

## Артефакты: что инструмент оставляет в мире

Артефакты — объекты эфемерного слоя (`document.scene`,
[справочник](scripting-api.md#эфемерный-слой-сцены-vrwebscene-objects1)); они живут
независимо от предмета и его realm, их видят все, включая поздно вошедших:

- **Универсальный артефакт — `kind="vrweb-node"`**: любой VRWML-тег с атрибутами
  (`{tag="VRWebImage", attrs={src=..., transform=document.values.to_literal(pose)}}`).
  Покрывает почти всё — панели, картинки, свет, меши-примитивы, даже вложенные grabbable.
  Значения атрибутов стройте через `document.values.to_literal`, а не склейкой строк.
- Типовые артефакты клиента: `vrweb-node` с тегом `VRWebStroke` (полилиния-труба:
  строковые attrs `points="[x,y,z,...]"`, `color="Color(r,g,b,1)"`, `width="..."`),
  `bubble`. Перечисление чужих артефактов — `document.scene.objects(kind)` (инструменты-
  модификаторы: ластики, клинеры).
- **Права уже есть**: действия идут от имени пользователя-держателя; править/удалять можно
  только свои объекты (чужие отклонит авторитет) — фильтруйте по `author` заранее.
- `ttl=0` — постоянный артефакт (кандидат на персистенцию), `ttl>0` — истекает сам.
- **Финализируйте, не стримьте**: копите локально (превью — `document.create`, мировая
  позиция — `set("global_position", ...)`), в слой шлите один `add` по завершении жеста.
- Офлайн работает без веток в вашем коде: слой становится локальной standalone-машиной.

## Файлы пользователя

`document.files.pick(kind, callback)` — системный диалог, выбор = согласие пользователя;
`kind` лишь фильтрует диалог. В callback приходит непрозрачный `file`-handle: чтобы артефакт
увидели другие, опубликуйте файл — `file.publish("image")` вернёт `vrwebblob://` URL, готовый
для `vrweb-node` (байты докачиваются пирами по требованию), причём крупное фото при этом не
втаскивается в память скрипта. Нужны сами байты (превью, разбор) — `file.bytes()`, он отдаёт
содержимое файлов до 2 MiB. Путь ОС скрипту не сообщается.

## Сеть: что синхронизировать самому

Бесплатно синхронизированы: **предмет в руке** (hold-состояние + attachment), **артефакты
слоя**, сам **факт существования предмета**. Всё остальное — обычные каналы автора
([scripting-patterns.md](scripting-patterns.md)):

- `use`/`use_end` — **локальные** события держателя. Если результат должен увидеть мир —
  держатель сам делает его каноном (`document.state.command`) или артефактом
  (`document.scene.add`); если результат мимолётный и адресный — `document.remote`.
- Правило то же, что всюду: поздно вошедший должен узнать → state/scene; не должен → remote.
- Состояние самого предмета (заряды, режим) — `document.state` со своей схемой; namespace
  realm'а уже делает его пер-экземплярным.

## Поверхность grabbable из скрипта

`handle.call` на своём носителе: `call("release", {})` — положить (только у держателя),
`call("holder", {})` / `call("held_hand", {})`, `call("set_enabled", {false})` — запретить
взятие (аналог `pickupable`). Полная таблица —
[scripting-api.md](scripting-api.md#grabbable-предметы-vrwebgrabbable1).

## Правила и лимиты (обязательное чтение)

- **⚠️ Полная арность host-вызовов**: передавайте все аргументы, необязательные — `nil`/`{}`
  (`document.scene.add(spec, nil)`, `call("play", {})`). Недостающие аргументы текущий
  биндинг не дополняет ([детали](scripting-api.md#эфемерный-слой-сцены-vrwebscene-objects1)).
- Лимиты realm — стандартные ([scripting-api.md → Ошибки и лимиты](scripting-api.md#ошибки-и-лимиты)):
  256 handles, бюджеты CPU/памяти, ошибка в callback закрывает realm предмета (сцена
  предмета остаётся). Клиент материализует до 32 item-realm'ов одновременно.
- `props` объекта слоя ограничены `MAX_PROPS_BYTES` — прореживайте данные (карандаш: шаг
  3 см, потолок 200 точек).
- Разметка предмета проходит content policy клиента наравне со страницами.
- `src` предмета иммутабелен: обновление инструмента = убрать объект и заспавнить новый src
  (версионируйте URL).

## Распространение и спавн

Item-документ — просто URL: свой хостинг, страница мира, бандл клиента
(`vrwebresource://items/...`). Спавн:

- **из скрипта страницы/другого предмета** — так мир раздаёт свои инструменты (стойка у
  входа). `position` — массив из трёх чисел (позиции в слое всегда JSON-сериализуемы):

  ```lua
  document.scene.add({ kind = "vrweb-item", parent = "", ttl = 0, props = {
    src = "https://example.org/tools/my-tool.html",
    position = { 2.0, 1.2, -3.0 },
  } }, nil)
  ```
- **клиентский тулбелт** ([client/tools.md](../client/tools.md)) — хоткеи стандартного
  набора; сторонние наборы — расширение клиента, не формата.

Убирается инструмент `document.scene.remove(id, nil)` — вместе со сценой и realm.

## Как тестировать

Headless-паттерн (референс — [tests/test_item_tools.gd](../../tests/test_item_tools.gd)):
`Settings.online_enabled = false` (standalone-слой), мир + пол + Player + `GrabManager` +
`EphemeralView` (с фейковым `file_picker`), спавн предмета `request_scene_action`, действия —
`manager.request_grab / use_held / use_held_end / release_held`, проверки — по
`NetworkManager.scene_objects()` и узлам сцены. Ошибки скрипта ловите сигналом
`script_failed` runtime'а.

## Чек-лист перед публикацией

- У `<VRWebGrabbable>` есть `id`; выбран режим `fixed`+`grip` или `adjustable`.
- Обработан `drop` (откат незавершённого действия).
- Все host-вызовы имеют полную арность (`nil`/`{}`).
- Артефакты финализируются одним `add` и укладываются в props-бюджет.
- Чужие объекты фильтруются по `author` до `remove`/`update`.
- Результат `use` уходит в state/scene, если его должны видеть другие.
- Два экземпляра предмета в одной комнате работают независимо.
- Офлайн-запуск не падает (standalone-слой).
