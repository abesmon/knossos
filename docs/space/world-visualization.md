# Визуализация мира из стилей документа

Как страница задаёт **палитру и небо** своего 3D-пространства. Цель: мир должен
визуально перекликаться с сайтом — тёмный сайт даёт тёмный мир, фирменный цвет фона
красит небо и стены, фоновая картинка `<body>` становится небом.

Связано: [html-to-3d-topology.md](html-to-3d-topology.md) (артефакт §1, фаза F),
[local-resources.md](../client/local-resources.md) (загрузка картинок), [clustering.md](clustering.md).

---

## 1. Что берём из документа

Из `<body>` вытаскивается **визуальный паспорт** — три необязательных поля:

| Поле | Источник CSS | Куда идёт в мире |
|---|---|---|
| `bg_image` | `background-image: url(...)` / `background: … url(…)` | **небо-панорама** (`PanoramaSkyMaterial`) |
| `bg` | `background-color` / `background` (цветовой токен) / атрибут `bgcolor` | **база неба** + тон палитры комнат |
| `fg` | `color` / атрибут `text` | **цвет земли** под горизонтом + тёмные акценты |

Правила визуализации (по запросу на отрисовку миров):

- если у `<body>` есть **фоновая картинка** — она берётся как **небо**;
- если картинки нет — берётся **`background-color`** и используется как база неба;
- **земля** (низ небосвода) — это **`color`** документа, а если его нет — **очень тёмный
  `background-color`**;
- **стены, полы и прочие элементы** красятся в тон документа: своя CSS-заливка контейнера —
  высший приоритет, иначе цвет выводится из фона `<body>` (а не из случайного сида).

## 2. Откуда берутся цвета: мини-каскад CSS

Паспорт собирает **мини-каскад** ([css-cascade.md](css-cascade.md)): внешние таблицы
(`<link rel="stylesheet">`) качаются `CssFetcher`'ом до фазы топологии, `StyleResolver`
применяет правила со специфичностью, наследованием и `var()`-переменными и кладёт
вычисленные стили в `node.computed`. `TopologyBuilder._document_style` читает паспорт из
computed самого `<body>` с фолбэком на `<html>` (фон часто объявлен на нём); `fg` берётся
только при **собственной** декларации `color` (`color-own`), а не унаследованной.
Презентационные атрибуты `<body bgcolor=… text=…>` перекрывают, как и раньше.

Инлайн `style` и `<style>`-правила — обычные участники каскада (инлайн — старший тир,
ниже только `!important`). Чтобы `<style>` дошёл до этой фазы, парсер **сохраняет его
содержимое** как текстовый узел (`HtmlParser`, ветка raw-text — рядом с `textarea`/`title`);
`<script>` по-прежнему выбрасывается.

Фолбэк: если резолвер не бежал (прямые вызовы `build` из тестов/дебага), работает старый
лёгкий экстрактор `_doc_style_from_stylesheet` (правила `body`/`html`/`:root` по порядку,
без специфичности) + инлайн `<body>` + атрибуты.

**Парсер цвета** на стороне геометрии (`WorldGenerator._parse_css_color`) понимает `#hex`
(включая `#rrggbbaa`, в котором топология сериализует вычисленные Color), именованные
цвета (`Color.from_string`) и `rgb()/rgba()` (альфа отбрасывается — миру нужен непрозрачный
цвет); `transparent`/нераспознанное → `null` (откат к следующему источнику/сиду).

## 3. Контракт

Паспорт кладётся в артефакт топологии рядом с `typography`:

```
Space {
  …
  document: { bg?: cssColor, bg_image?: url, fg?: cssColor }   # все поля опциональны
}
```

Хранятся **сырые строки** CSS — геометрия (фаза F) сама их парсит и решает, что делать.
Это держит границу топология↔геометрия: топология не знает про `Color`/панорамы.

## 4. Как это рисуется (фаза F, `WorldGenerator`)

`_resolve_document_palette()` (зовётся первой в фазе атмосферы) выводит:
- `_doc_base_color` / `_doc_has_base` — база неба и палитры: цвет `document.bg`, а если его
  нет — **средний CSS-фон комнат страницы** (старое поведение как запасной вариант);
- `_doc_fg` — цвет текста `<body>` или `null`.

**Небо** (`_build_atmosphere`):
- есть `bg_image` → асинхронно грузим текстуру через `ImageLoader` и подменяем небо
  **панорамой** (`_apply_sky_image`); до прихода текстуры висит цветное небо, чтобы мир был
  освещён сразу (стриминг, [performance-streaming.md](../client/performance-streaming.md)). Картинка
  трактуется как равнопрямоугольная — обычная картинка растянется, но «как небо» работает;
- иначе → процедурное небо (`ProceduralSkyMaterial`): верх = `bg` как есть (тёмный фон → тёмное
  небо), горизонт чуть теплеет к закату. Без цвета — процедурная палитра от тона/сида (старое
  поведение).

**Земля** (низ небосвода `ground_bottom_color`): `_doc_fg`, иначе очень тёмный `bg`
(`darkened(0.85)`), иначе старый тёмный тон от сида.

**Комнаты** (`_room_color`): свой `hints.css.bg` контейнера → как есть; иначе оттенок от
`_doc_base_color` (соединители — затемнённый базовый; комнаты — вариация тона по `id`);
без цвета документа — прежний случайный оттенок. Стены = цвет пола, осветлённый.

Высота солнца/насыщенность по-прежнему зависят от «богатства» контента (вес корня) и сида URL
— детерминированно: один сайт = одно небо.

## 5. Дальше (нереализовано)

  ([css-cascade.md](css-cascade.md)) качает внешние CSS до фазы топологии и применяет их

  стены комнаты), а не только цвет.

  настоящим веб-движком; мини-каскад их принципиально не даёт.
