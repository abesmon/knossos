# VRWeb scripting v1

> Статус: реализованный MVP и целевой формат первой версии. Скриптинг не является частью
> Godot/VRWML object model и не использует прикреплённые GDScript, кодовые компоненты или пакеты.

VRWeb использует интерпретируемый Luau как переносимый язык поведения страницы. WASM не требуется
для этой версии и позже может появиться как отдельный, более тяжёлый runtime profile.

## Объявление в HTML

Скрипты выполняются в порядке документа. `id` обязателен и уникален в пределах страницы.

```html
<script type="application/vrweb+luau" id="room.lights">
local lamp = document.query("#lamp")
if lamp then
  lamp.on("activate", function(event)
    lamp.set("visible", not lamp.get("visible"))
  end, "Switch light")
end
</script>

<script type="application/vrweb+luau" id="room.chat"
        src="./chat.luau"
        integrity="sha256-BASE64"></script>
```

Ровно одна форма обязательна:

- inline: исходный текст внутри элемента;
- linked: `src`, разрешаемый относительно URL документа.

`src` и непустое тело одновременно являются ошибкой. Клиент получает и интерпретирует исходник;
страница не поставляет bytecode. Профиль семантики runtime: `vrweb-luau/1`.

Knossos загружает linked source с таймаутом 15 секунд и не более чем через 8 redirects. HTTP(S)
документ не может ссылкой прочитать `vrweblocal://` или `vrwebresource://`: локальная файловая
схема доступна только локальному документу, а bundle-схема — документу из того же bundle.
Политика downgrade и проверка каждого redirect hop остаются отдельным transport-hardening слоем.

## Integrity

Поведение совпадает с opt-in моделью HTML Subresource Integrity:

- нет `integrity` — проверка не запрошена, скрипт разрешено загрузить;
- `integrity` есть — клиент обязан проверить поддерживаемый digest до исполнения;
- digest не совпал или алгоритм не поддержан — этот скрипт не исполняется.

MVP поддерживает `sha256-BASE64`. Integrity подтверждает соответствие полученных байтов ожиданию
автора документа, но не выдаёт скрипту полномочий и не доказывает безопасность кода.

## Realm и жизненный цикл

Каждый page script получает отдельную sandboxed Luau VM и отдельный `document.session`. В VM нет
Godot objects, файловой системы, ОС, сети, загрузки нативных библиотек и произвольного доступа к
объектам клиента. Доступны безопасные стандартные библиотеки Luau и единственный host object
`document`.

Навигация уничтожает все realms страницы. Ошибка загрузки или top-level исполнения одного скрипта
не откатывает декларативную сцену и не запускает его staged side effects. Callback вызывается
через контролируемый вход VM с interrupt deadline; ошибка или превышение CPU budget отключает
realm вместе с его handlers, timers и owned objects, не обрушая клиент.

## Realtime replacement

Клиент может заменить активный скрипт по `id`, передав source и hash активной базовой ревизии.
Новая VM сначала компилируется и исполняется в staging. Только успешная ревизия атомарно заменяет
handlers/timers/owned objects старой. При ошибке старая продолжает работать. Сериализуемый
`document.session` переносится в новую VM; состояние сцены и distributed state живут вне VM.

Это позволяет редактировать поведение во время realtime-сессии без перезагрузки пространства.
Транспорт и право публикации ревизии не входят в scripting v1.

## Слои безопасности

Модель намеренно разделена на четыре пересекающихся слоя:

1. стандартный capability API VRWeb;
2. фактически реализованный и ограниченный клиентом набор;
3. разрешения пользователя, сужающие клиентский набор;
4. правила доступа конкретного instance.

MVP реализует первые два. Третий и четвёртый остаются семантическими точками расширения и пока не
сужают доступ. Это та же модель, что применяется к поддержке VRWML-тегов: документ может запросить
возможность, реализация объявляет/разрешает реально доступное пересечение.

CPU deadlines, memory ceilings, лимиты handles/timers/calls и допустимые engine classes —
обязательная ответственность безопасного клиента, а не переносимое значение стандарта. Knossos
применяет собственные консервативные лимиты; жёсткий allocator ceiling может быть добавлен в
GDExtension отдельно, не меняя HTML или `document` API.

## Совместимость реализаций

Стандарт фиксирует Luau-синтаксис, профиль библиотек, форму объявления, `document` API и наблюдаемую
семантику. Godot-типы не передаются напрямую. Knossos отображает opaque handles на Godot nodes;
другой клиент отображает те же операции на собственную сцену. Новые возможности добавляются как
versioned capability names, не расширением неограниченного engine bridge.

API описан в [scripting-api.md](scripting-api.md).
